# Wait, We Don't Need to "Wait"! Removing Thinking Tokens Improves Reasoning Efficiency

- https://arxiv.org/abs/2506.08343

## Summary

### 배경
- 최근 LRM (Large Reasoning Model)이 발달함 (DeepSeek R1 등)
- five R1-style model series (DeepSeek R1 style)
  - RL에서 모델이 스스로 문제를 풀고 "이게 맞나?"를 고민하며 보상을 받음
  - chain-of-thought: 사용자 질문에 바로 답을 내놓는 것이 아니라 복잡한 추론 과정을 거침 (think 태그로 감싸진 긴 텍스트가 포함되는 것이 특징)
    - Aha Moment phenomenon: reasoning trajectory with anthropomorphic expressions ("wait", "hmm", "alternatively" 등)
    - $CoT = {(chunk_i, a_i)}$
  - self-correction: 중간에 계산이 틀리거나 논리가 꼬이면 스스로 알아차리고 수정
 
  
- overthinking problem 대두:
  - verbose(장황한) reasoning
  - redundant(불필요한) thought steps
  - -> computational overhead, high reasonoing latency 초래 -> computational cost 증가

- overthinking problem 해결을 위한 현행 연구
  - additional training을 필요로함 (RL, fine-tuning)
  - training-free approach: 성능 저하 발생

### 연구 내용
- "Wait"과 "Hmm"과 같은 explicit self-reflection 토큰의 필요성?
- "NoWait" 방법론 제안
  - training-free 
- self-reflection 토큰을 생성을 억제하기 위한 three main stages 
  - initialize reflection keywords list 생성 (예: “Wait”, “Alternatively”, “Hmm” 등)
  - special token-level keyword list 생성 (variants) (예: wait -> "Wait", ".wait", "WAIT")
  - suppressing keywords generation: 이 키워드들에 large negative value 부여 (inference time의 decoding에 이 토큰들의 logit에 아주 큰 음수값 부여) 


### 실험
- models(R1 style) and benchmarks
  - textual reasoning: RL based models
  - visual reasoning: SOTA RL based visual models
  - video reasoning: QvQ-72B-Preview
- Metrics: accuracy, generation length
- Baselines: model's original performance, NoThink strategy (NoWait이랑 다른 점: 프롬프트 엔지니어링으로 directly remove the entire reasoning process)


### Results
- chain-of-thought trajectory legnth를 27-51% 줄임 
- model utility 손상 없음  
- Plug and Play (PnP): 꽂으면(Plug) 바로 실행된다(Play)
  - 추가 학습(Training)이 필요 없음
  - 기존 구조를 바꾸지 않음
  - 즉각적인 도입 가능
- Model Architecture 일반화 잘됨
- Multimodal Reasoning 잘됨
- distillation model은 성능 저하 발생


## Opinion
- overthinking problem에 관한 첫 논문으로서 개괄적인 지식을 얻었다.
- 원래 naive 하게 생각했었을 때는 text length와 모델의 답변 성능이 정비례(ㅋㅋ)할 거라고 생각하고 있었는데, 그렇지 않음을 알게 되었다. (조금만 생각해봐도 아닐텐디 ㅜㅜ)
- overthinking problem 뿐만 아니라 reasoning 시에 발생하는 다양한 문제들에 대해 '로짓 값 조절'을 통한 해결책을 도입할 수 있을 것 같다.
