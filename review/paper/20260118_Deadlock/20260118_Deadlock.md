# One Token Embedding Is Enough to Deadlock Your Large Reasoning Model

- https://arxiv.org/abs/2510.15965

## Summary

### 연구 배경
- overthinking에 대한 정의 (더 이상의 자세한 설명은 생략한다.) 

- 기존 해결 방법
  - Training-centered methods
    - RL에 novel reward function
    - apply SFT on datasets
  - Output-based methods
    - compressing reasoning trajectories in latent space
  - Input-based methods
    - using explicit prompts to elicit concise reasoning

- 기존 공격 방법론
  - 잘못된 출력 유도 (정확성 손상)
  - jailbreaking -> harmful content 생성 -> undermine safety
  - backdoor attack -> involves implanting triggers during training time (**이 논문에서 다루는 것**)
  - slowdown attack 

### 연구 내용
- **Deadlock Attack**
  - training a malicious adversial embedding (악의적인 적대적 임베딩)
  - induce perpetual reasoning loops (영원한 추론 루프,,,)

- white-box access to the victim LRM

- token의 작동 방식: end-of-thought punctuation (. ?) 바로 다음에 reflective/hesitant tokens 생성 확률을 극대화 (wait, but)
- input agnostic (입력에 무관함)

- 과정
  - 1단계: optimizing a universal adversarial embedding (연속 embedding space에서)
  - 2단계: 위에서 작업한 것을 discrete backdoor trigger에 mapping (adversarial embedding을 덮어 씌우는 것) -> implant this trigger into an open-source LRM
    - 이렇게 하는 이유? -> continuous-to-discrete projection gap 때문에
      - attack을 현실에 적용할라면: continuous adversarial embedding -> discrete token sequence (in model's V) 로 바꾸는 과정이 필요함
      - 문제: 근데 원래는 adversaial embedding을 가장 가까운 embedding으로 projection하는 과정에서 공격이 **무효화** 됐었음
      - 해결 방법: backdoor mechanism (optimized adversarial vector를 미리 정의된 embedding matrix에 직접 넣음)
    - 장점
      - 효율성: 모델 전체 미세 조정 x 소수 임베딩만 학습
      - 은밀성: trigger token이 입력에 없을 때는 다른 기능들에서 티가 안 남 (걸릴 가능성 적음)

- backdoor에 접근에 대해서 더 찾아봄 (출처 제미나이)
  - A. 공급망 공격 (Supply Chain Attack): 가장 흔한 시나리오입니다. 허깅페이스(Hugging Face) 같은 곳에 올라온 '성능 좋은 오픈소스 모델'인 척하며, 이미 백도어가 심어진 모델 파일을 배포하는 것입니다. 사용자가 이 모델을 다운로드해 사용하는 순간 공격자에게 노출됩니다.
  - B. 데이터 오염 (Data Poisoning): 모델을 미세 조정(Fine-tuning)할 때 사용하는 데이터셋에 특정 패턴을 심어두어, 학습 과정 중에 모델이 스스로 특정 단어를 공격 벡터와 연결하도록 유도합니다.
  - C. 가중치 직접 수정 (Weight Manipulation): 공격자가 모델이 저장된 서버나 스토리지에 권한을 얻었을 경우, model.safetensors나 pytorch_model.bin 같은 가중치 파일에서 임베딩 레이어의 특정 행(row)만 살짝 수정합니다.
