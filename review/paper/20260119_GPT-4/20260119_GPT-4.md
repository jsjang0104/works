## 1 요약

### 1.1 Key Idea

#### 1.1.2 GPT-4의 특징
- 이전보다 더 큰 scale (이전보다 정확히 몇배 scaling된 건지는 OpenAI가 open하지 않음)
- multimodal (input: image, text -> output: text)
- various human-level performances
- pre-trained Transformer based next token prediction model -> post-training alignment process

#### 1.1.3 Key Challenge: Predictable Scaling
- infrastructure & optimization method를 고도화 -> 1/1000 수준의 컴퓨팅으로 학습시킨 작은 모델 결과로도 GPT-4와 같은 큰 모델의 성능을 예측할 수 있게 됨
- LLM의 final loss는 사용된 컴퓨팅 양에서의 power laws로부터 근사할 수 있음  
  - power laws: computing power와 성능은 비례한다는 규칙
  - 성능 예측 공식 : ${L(C) = aC^b + c}$
    - L(C): final loss
    - C: 투자한 computing power
    - a,b: 학습 효율 결정 변수
    - c: 아무리 학습시켜도 줄일 수 없는 최소한의 오차 (irreducible loss)
- GPT-4 학습 초기에 이 법칙으로 근사하여 예측을 해놨었는데, 나중에 확인해보니 그게 정확했다는 것 (소름끼치게 똑같음. 밑 Figure 확인)

### 1.2 실험 및 결과
- 원래 인간용인 실험으로 test 되었음 (NLP 능력 테스트)
  - simulated bar exam (미국 변호사 시험): test taker들 중 상위 10% 차지 (GPT-3.5 시절에는 하위 10%였음)

- Traditional NLP benchmarks에서도 기존 LLM과 SOTA들을 모두 압도
  - 영어뿐 아니라 다른 언어들에서도 성능 좋음

## 2 의견
- 모델 자랑만 하고 정확히 그래서 어떻게 한 건지는 안 알려주니 궁금하네요
- 그래서 제미나이한테 물어봤는데 이렇게 추정된다고 합니다:

1. Mixture of Experts (MoE, 전문가 혼합 구조): 가장 유력한 설은 GPT-4가 하나의 거대한 덩어리가 아니라, 여러 개의 전문 모델을 합쳐놓은 구조라는 것입니다.

- 어떻게 했나: 약 2,200억 개의 파라미터를 가진 모델 8~16개를 하나로 묶어 총 1.8조 개 정도의 파라미터를 가졌을 것으로 추정됩니다.

- 왜 했나: 질문이 들어올 때마다 1.8조 개를 다 쓰는 게 아니라, 질문 내용에 맞는 '전문가 모델' 2개 정도만 활성화해서 답변합니다. 이렇게 하면 지능은 엄청나게 높이면서도, 연산 비용과 속도는 효율적으로 관리할 수 있습니다.

2. 압도적인 학습 데이터의 양과 질: GPT-3가 '인터넷에 있는 글을 닥치는 대로' 읽었다면, GPT-4는 훨씬 정교하게 선별된 데이터를 먹었습니다.

- 데이터 양: 약 13조 개의 토큰(단어 조각)을 학습한 것으로 알려져 있습니다 (GPT-3의 약 26배).

- 멀티모달 학습: 텍스트뿐만 아니라 이미지 데이터를 함께 학습시켜 사물에 대한 시각적 이해력을 갖게 했습니다.

- 코드 데이터: 코딩 능력을 키우기 위해 전 세계의 수많은 프로그래밍 코드를 집중적으로 학습시켰는데, 이것이 모델의 전반적인 논리적 추론 능력까지 끌어올리는 효과를 냈습니다.

3. '예측 가능한 확장'을 위한 엔지니어링: 질문하신 논문 내용의 핵심이기도 한데, 이들은 **"실패하지 않는 학습 시스템"**을 구축했습니다.

- 보통 모델이 너무 커지면 학습 도중에 계산이 꼬이거나(Loss Spike) 성능이 안 나오는 경우가 허다합니다.

- OpenAI는 작은 모델로 수만 번 테스트하며 '돈을 이만큼 쓰면 성능이 저만큼 나온다'는 수학적 공식을 완벽히 검증한 뒤에 GPT-4 학습 버튼을 눌렀습니다. 즉, "수천억 원짜리 도박"을 "확실한 투자"로 바꾼 엔지니어링 기술이 진짜 자랑거리인 셈입니다.
